## Repository Layout

This project is organized into the following folders:

- **data/**: Contains all data used in the project *(ignored by version control)*.
  - **simulations/**: Raw data generated by simulations.
  - **experimental/**: Raw data collected from real-world experiments.
  - **processed/**: Cleaned and processed data for analysis.
  - **tmp/**: Temporary or intermediate data files.
- **logs/**: Log files generated during script execution *(ignored by version control)*.
- **results/**: Analysis results like figures, CSVs, or other outputs *(ignored by version control)*.
- **scripts/**: Scripts for data processing, analysis, and visualization.
- **src/**: Source code, reusable modules, or libraries.
- **_research/**: Research notes, exploratory analysis, and supporting documentation.
  - **tmp/**: Temporary or intermediate research files *(ignored by version control)*.

## Important Notes

1. **Temporary and Data Files**: Files in the following folders are ignored by version control:
   - `data/` (except for its `README.md` file)
   - `logs/`
   - `results/`
   - `_research/tmp/`

2. **Data Management**:
   - Keep raw data immutable. Do not edit files in `data/simulations/` or `data/experimental/`.
   - Use `data/processed/` for cleaned or aggregated data.

3. **Logging and Results**:
   - Use descriptive names for log files and results to keep them organized (e.g., `process1_20241202.log`).
   - Archive old logs and results when no longer needed, or use `make clear` to clear all temporary, logs and result files (be careful!).
